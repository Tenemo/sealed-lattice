Use the root package for the full supported public surface. The package does
not ship committed public subpath entry points at this stage.

## What lives here

- `sha256Hex`
- `UnsupportedRuntimeError`

## Common starting points

- digest text or bytes: `sha256Hex`
- fail closed on missing native support: `UnsupportedRuntimeError`

## Typical import style

```typescript
import { UnsupportedRuntimeError, sha256Hex } from "sealed-lattice";

const digest = await sha256Hex("sealed-lattice");
```

## Use the root package when

- you want the supported SHA-256 helper
- you want the typed runtime failure for missing Web Crypto digest support
- you want the stable public surface without depending on future internal structure

## What stays intentionally internal

- lattice encryption and threshold flows
- serialization and transport boundaries
- proof systems
- protocol types
- any future public subpath layout
